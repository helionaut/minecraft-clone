# Inventory Icon Asset Pack

This document replaces the HEL-109 placeholder checklist with the final inventory icon asset spec for the current production UI.

## Final Art Decision

- Asset source: repo-authored Minecraft-style SVG icons generated by [`scripts/generateInventoryIcons.mjs`](/home/helionaut/workspaces/HEL-111/scripts/generateInventoryIcons.mjs)
- Prepared outputs: checked-in `16x16` transparent SVG files under [`public/textures/inventory/`](/home/helionaut/workspaces/HEL-111/public/textures/inventory)
- Implementation mapping: [`src/ui/inventoryIcons.ts`](/home/helionaut/workspaces/HEL-111/src/ui/inventoryIcons.ts)
- UI consumption: [`src/ui/createAppShell.ts`](/home/helionaut/workspaces/HEL-111/src/ui/createAppShell.ts) applies each icon as a slot-safe background image
- Licensing posture: no external marketplace or third-party asset dependency; the pack is produced in-repo from deterministic source code

## Pack Rules

- Every icon uses a transparent background and a `16x16` pixel grid tuned for hotbar and inventory-slot readability.
- Blocks render as square tile faces; tools and materials render as centered silhouettes with transparent margins.
- Visual language follows Minecraft item-family cues without requiring any external asset source.
- Regeneration command: `npm run generate:inventory-icons`
